Подробное описание

Раздел классов доступа к настройкам программы.

Работа с настройками

  • Программа при запуске дожна создавать нужные ей для работы параметры при помощи функции alicorn::modules::settings::Section::SetDefault():
    ::alicorn::modules::settings::Section(uT("Root")).SetDefault(uT("Name"), 12345, uT(""));
    Отсутствующие в файле настроек параметры будут созданы с указанными значениями, прочитанные из файла параметры переписаны не будут.
  • Созданные параметры при завершении работы программы будут сохранены в файл настроек со значениями по умолчанию и могут быть изменены при помощи утилиты Settings.App.
  • Значения параметров могут быть получены при помощи функции Get<Type>(Name):
    const auto Param = ::alicorn::modules::settings::Section(uT("Root")).Get<int>(uT("Name"));
  • Порядок поиска файла настроек см. в описании конструктора класса alicorn::modules::settings::File.

Особые параметры

  • Программа при запуске может создать используемые в значениях параметров особые параметры (функция alicorn::modules::settings::Section::AddExtra()); эти параметры не сохраняются в файл настроек и должны создаваться заново при каждом старте программы (предполагается, что в качестве особых параметров будут использоваться параметры, значения которых зависят от конкретного запуска, например, параметр ModuleLocation (он автоматически создается всегда!) - путь к запускаемой программе).
    ::alicorn::modules::settings::Section(uT("Root")).AddExtra(uT("Extra"), 12345);
  • Особые параметры могут быть созданы только один раз, в дальнейшем изменить их будет нельзя.
  • Особые параметры будут автоматически подставляться вместо тегов вида {ParamName} при запросе значений параметров.
    Например, если параметр с именем PathToData содержит значение "{ModuleLocation}\Data", а текущая программа расположена в папке "C:\Program Files\MyProgram", то при запросе значения параметра PathToData функция вернет "C:\Program Files\MyProgram\Data".
  • Поиск особого параметра начнется с текущего раздела и при его отсутствии продолжится рекурсивно вверх по иерархии разделов, поэтому особые параметры логичнее всего создавать в корневом разделе настроек.

Особые параметры, автоматически создаваемые библиотекой

  • ModuleLocation - путь к запускаемой программе.

Разделы

  • Параметры программы могут быть разнесены по разделам, которые могут быть организованы в виде иерархического дерева.
    Получение параметра раздела:
    const auto Param = ::alicorn::modules::settings::Section(uT("Root"))[uT("PartName")].Get<int>(uT("Name"));
    или
    auto Part = ::alicorn::modules::settings::Section(uT("Root"))[uT("PartName")];
    const auto Param = Part.Get<int>(uT("Name"));
  • Разделы могут наследовать параметры других разделов. Это можно использовать в случаях, когда большинство параметров не зависят от режима запуска программы, но есть параметры, которые должны иметь иметь разные значения в зависимсоти от режима запуска. В этом случае создается иерархия разделов, в которой общий раздел содержит независимые параметры и несколько разделов со специфическими параметрами, наследующими параметры общего раздела. Программно такую иерархию создать нельзя, поэтому нужно предварительно создать файл настроек, содержащий нужную иерархию и положить его в папку программы при первом запуске.
    Пример такого файла настроек:
    <?xml version="1.0" encoding="utf-8"?>
    <root>
    <settings>
    <Unknown parent="" />
    <Vs2008 parent="Unknown" />
    <Vs2013 parent="Unknown" />
    </settings>
    </root>
    Теперь при создании параметров в разделе Unknown они будут доступны также и в разделах Vs2008 и Vs2013. Параметры, созданные в двух последних разделах, будут доступны только в них (параметры с теми же именами, что и в унаследованном разделе будут перезаписаны).
    Наследоваться будут также и подразделы, если таковые будут присутствовать.

Примеры использования

Образец файла настроек:

<?xml version="1.0" encoding="utf-8"?>
<root>
<!-- Раздел настроек -->
<settings>
<!-- Общие настройки для всех остальных разделов -->
<Common parent="">
<IsTrue comment="Bool value.">true</IsTrue>
<IsFalse>false</IsFalse>
<English comment="Text value.">English</English>
<Russian>Русский</Russian>
<String>Empty</String>
<Number>123</Number>
<Reference1>Number: {Number}</Reference1>
<Reference2>Param: {Unknown}</Reference2>
<Reference3>Number: {Number} {Number} {String}</Reference3>
<Reference4>Number: {Number} {Unknown} {String}</Reference4>
<Param1>Unknown</Param1>
<First parent="">
<Param1>First</Param1>
</First>
<Second parent="">
<Param1>First</Param1>
<Param2>Number: {Number} {Param1}</Param2>
</Second>
</Common>
<!-- Раздел, ссылающийся на раздел Common (существующие в разделе -->
<!-- параметры будут загружены как есть, после чего будут добавлены -->
<!-- отсутствующие параметры из родительского раздела). -->
<Current parent="Common">
<Number>234</Number>
</Current>
<!-- Раздел, ссылающийся на раздел Current -->
<Adding parent="Current">
<String>Using string</String>
<Int>12345</Int>
</Adding>
<Empty parent=""/>
<Empty2 parent="Common"/>
</settings>
</root>

Работа с параметрами:

// Создание объекта раздела Adding
auto AddingSection = ::alicorn::modules::settings::Section(uT("Adding"));
// Создание объекта подраздела Adding\Second
// cppcheck-suppress unreadVariable
auto AddinSecondSection = AddingSection[uT("Second")];
// Создание нового раздела
auto AddinNewSection = AddingSection[uT("New")];
//
// Установка значений по умолчанию
AddingSection.SetDefault(uT("Bool"), true, uT("Bool value")); // не существует, будет добавлен
AddingSection.SetDefault(uT("Bool2"), uT("true")); // не существует, будет добавлен
AddingSection.SetDefault(uT("Number"), 999); // уже существует, не изменится
//
AddinNewSection.SetDefault(uT("One"), 1);
AddinNewSection.SetDefault(uT("Two"), uT("2"));
AddinNewSection.SetDefault(uT("Three"), uT("333"));
//
// Чтение особых параметров
EXPECT_EQ(::alicorn::system::application::CurrentModule::GetLocation(),
AddingSection.Get<Path_t>(uT("ModuleLocation")));
//
// Чтение существующих параметров
EXPECT_EQ(true, AddingSection.Get<bool>(uT("IsTrue")));
EXPECT_EQ(false, AddingSection.Get<bool>(uT("IsFalse")));
EXPECT_EQ(234, AddingSection.Get<int>(uT("Number")));
EXPECT_EQ(12345, AddingSection.Get<int>(uT("Int")));
EXPECT_EQ(true, AddingSection.Get<bool>(uT("Bool"))); // добавленный
EXPECT_EQ(true, AddingSection.Get<bool>(uT("Bool2"))); // добавленный
EXPECT_EQ(uT("Using string"),
AddingSection.Get<String_t>(uT("String")));
EXPECT_EQ(1, AddinNewSection.Get<int>(uT("One")));
//
// Чтение параметров, содержащих ссылки на другие параметры
EXPECT_EQ(uT("Number: 234 234 Using string"),
AddingSection.Get<String_t>(uT("Reference3")));
EXPECT_EQ(uT("Number: 234 {Unknown} Using string"),
AddingSection.Get<String_t>(uT("Reference4")));
//
// Чтение несуществующих параметров (исключение)
EXPECT_THROW(AddingSection.Get<String_t>(uT("Unknown")), ::std::exception);
EXPECT_THROW(AddinNewSection.Get<String_t>(uT("Five")), ::std::exception);
//
// Чтение с указанием невалидного типа (исключение)
EXPECT_THROW(AddingSection.Get<bool>(uT("String")), ::std::exception);
EXPECT_THROW(AddingSection.Get<int>(uT("String")), ::std::exception);
//
// Добавление особого параметра
AddingSection.AddExtra(uT("ExtraName"), uT("ExtraValue"));
EXPECT_THROW(AddingSection.AddExtra(uT("ExtraName"), uT("")), ::std::exception); // Нельзя добавить существующий параметр
EXPECT_THROW(AddingSection.AddExtra(uT("IsTrue"), uT("")), ::std::exception); // Нельзя добавить существующий параметр
EXPECT_EQ(uT("ExtraValue"),
AddingSection.Get<String_t>(uT("ExtraName")));
//
AddingSection.AddExtra(uT("ExtraName2"), 12345);
EXPECT_EQ(uT("12345"),
AddingSection.Get<String_t>(uT("ExtraName2")));
//
const String_t ExatraValue = uT("Value1512222258");
AddingSection.AddExtra(uT("ExtraName3"), ExatraValue);
EXPECT_EQ(ExatraValue,
AddingSection.Get<String_t>(uT("ExtraName3")));
//
// Существующий пустой раздел
auto EmptySection = ::alicorn::modules::settings::Section(uT("Empty"));
EXPECT_THROW(EmptySection.Get<String_t>(uT("IsTrue")), ::std::exception);
EmptySection.SetDefault(uT("IsTrue"), true, uT("")); // не существует, будет добавлен
EXPECT_EQ(true, EmptySection.Get<bool>(uT("IsTrue")));
//
// Существующий пустой раздел, наследующий настройки другого раздела
auto EmptySection2 = ::alicorn::modules::settings::Section(uT("Empty2"));
EXPECT_EQ(true, EmptySection2.Get<bool>(uT("IsTrue")));
