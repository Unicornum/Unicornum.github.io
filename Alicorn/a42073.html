Описание раздела

Раздел для вспомогательных проектов, собирающих проекты отдельных библиотек.

Проект Alicorn

Проект сборки всех проектов, предназначенных для использования в Windows.

Проект Android

Проект сборки всех проектов, предназначенных для использования под Android.

Заметки
Понадобился из-за того, что если Android-проекту добавить в зависимость Windows-проект, то проваливается сборка скриптом всего решения (хотя сборка в Visual Studio работает).

Проекту Test.Android/SourceVfsData (который формирует файл .pack, упаковываемый в .apk проектом Test.Android/Packaging) требуется зависимость от Test.Android/PackBuilder'a (который является Windows-проектом, т.к. должен запускаться при сборке решения), что и приводит к ошибке сборки скриптом. В качестве временного решения используется проект Android, который строится отдельной командой в BuildAll.cmd после построения всего остального.

Проект BuildAll

Проект, предназначенный для сборки всех остальных проектов 'одним кликом'.

  • Должен зависеть ото всех остальных проектов раздела Build и Test.Solution.
  • Используется для автоматической сборки скриптом BuildAll.cmd.
Заметки
Проект BuildAll понадобился потому, что сборка при помощи MsBuild всего решения приводит к сборке выгруженных проектов.
Предупреждения
10.05.2016: Параметры MsBuild '/p:DebugType=none;DebugSymbols=false' не работают для проектов lib, оставлены для ускорения сборки exe и dll.

Проект Clean

Проект предназначен для очистки временной папки перед началом сборки решения скриптом.

Заметки
Понадобился из-за того, что при работе с отдельными тестовыми проектами классов временная папка разбухает от временных файлов, после чего ночная сборка завершается ошибкой из-за недостаточного места на диске.
