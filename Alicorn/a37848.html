Файлы свойств

Solution.props

Надстройка Properties (входящая в UniCA) подключает к каждому проекту решения одну из страниц свойств из папки $(SolutionDir)/Solution (путь к папке задается как параметр PathToSolutionProps надстройки в разделе VisualStudio; там же можно изменить порядок поиска файлов):

  • Solution.$(Config).$(Platform).props
  • Solution.$(Platform).$(Config).props
  • Solution.$(Platform).props
  • Solution.$(Config).props
  • Solution.props

Эта страница свойств может содержать свойства, специфичные для данного решения и должна включать в качестве дочерних:

  • Externals/Windows.props или Externals/Android.props.
  • Страницу свойств используемых библиотек
    • Externals/Alicorn/Alicorn.props (включает страницы свойств используемых сторонних библиотек).
    • Externals/Covellite/Covellite++.props (включает страницу свойств Alicorn.props).

Default.props

Страница свойств, содержащая настройки проектов по умолчанию (включена в Externals/Alicorn/Requirements/Requirements.props).

Определяет макросы:

  • $(IsBinary): 'Yes' для проектов Exe и Dll и 'No' для Lib.
  • $(IsTest): 'Yes' для проектов тестирования и 'No' для всех остальных.
  • $(IsNotTest): 'No' для проектов тестирования и 'Yes' для всех остальных.
  • $(IsAndroid): 'Yes' для проектов Android и 'No' для всех остальных.
  • $(ProjectShortName): имя проекта без суффикса.
  • $(PartName): имя раздела проекта (состоит из имен папок между папкой решения и проекта, разделенных точками).
  • $(ResultType): 'Bin' для проектов Exe и Dll и 'Lib' для проектов Lib.
  • $(TypeProjectDir): имя вложенной папки для результата построения проекта ('Bin' для Exe и Dll, 'Lib' для Lib и 'Test' для проектов тестирования).
  • $(Prefix): префикс для файла результат построения (пустая строка для бинарных не тестовых проектов и имя раздела для всех остальных).
  • $(PrefixAndroid): 'lib' для проектов статических библиотек Android и пустая строка для всех остальных.
  • $(Suffix): пустая строка для Bin и Dll и имя конфигурации для Lib.
  • $(BinaryRootDir): корневая папка построения проектов Bin и Dll.
  • $(PlatformToolsetDirectory): 'vs140' для VS2015, 'gcc_4_9' для GCC 4.9, 'clang_3_8' для Clang 3.8 и т.п.
  • $(ResultDir): папка результата построения для проекта (папка решения + папка конфигурации (Debug или Release) + тип проекта (Bin, Lib или Test) + имя платформы (x86, x64 или arm)).
  • $(FullSolutionName): имя решения, созданное из полного пути решению без имени диска, в котором слеши заменены на символ подчеркивания.
  • $(VsBuildDisk): имя диска, на котором будет располагаться папка временных файлов построения проектов решения.
  • $(VsBuildDir): путь к папке временных файлов построения проектов решения.

Устанавливает свойства:

  • Путь к выходному каталогу.
  • Путь к промежуточному каталогу.
  • Имя файла результата (к проекту статической библиотеки добавляется имя раздела и имя конфигурации).
  • Пути '$(SolutionDir)' и '$(SolutionDir)\Solution\Solution' в качестве дополнительных каталогов включения заголовочных файлов.
  • Включены проверки SDL.
  • Включено использование полных путей.
  • Отключена инкрементная компоновка.
  • Параметр 'Компоновать библиотеки из зависимостей' установлен в 'Нет' для того, чтобы exe-шники, зависящие от dll не требовали соответствующих .lib файлов.
  • Включена оптимизация для отладки.
  • Создание .pdb файла в папке промежуточного результата.
  • Заданы увеличенные резервируемый и фиксируемый размеры стека.

Windows.props и Android.props

Страницы свойств, содержащие специфические настройки, присущие только Windows или Android проектам.

Windows.props:

  • Формат отладочной информации установлен в 'База данных программы', чтобы убрать предупреждение 'warning D9007: для "/Gm" требуется "/Zi или /ZI"'.
  • Включен 4-й уровень предупреждений (в создаваемых проектах наследовать свойство от родителя явным образом, т.к. по умолчанию установлен 3-й уровень предупреждений).
  • Игнорирование предупреждений 4099.

Android.props:

  • В качестве каталога включения добавлен путь Externals/Android (в котором расположен пустой файл SDKDDKVer.h) для того, чтобы в проекты Android можно было включать файлы, содержащие ссылки на файл stdafx.h).
  • Включена информация о типах времени выполнения.
  • Включены исключения С++.
  • Включено компилирование кода как кода С++.
  • Включена поддержка стандарта языка С++14.
  • Отключен формат отладочной информации.
  • Включено использование всего архива.
  • Добавлен путь поиска библиотек.

Externals.Build.props

Страница свойтв, предназначенная для подключения к проектам библиотек, которые должны собираться в папку Externals.

  • Выходной каталог настроен таким образом, чтобы построение производилось в папку с именем библиотеки внутри папки Externals/$(LibraryName)/lib (макросу $(LibraryName) должно быть установлено значение имени библиотеки на странице свойств Solution.props решения).
  • Перед сборкой проекта копируются все .hpp, .inl и .rc файлы из проекта (включая вложенные папки) в папку Externals/$(LibraryName)/include/$(LibraryName).
  • Перед сборкой проекта (в папку include/$(LibraryName)/lib-info) генерируется файл подключения собранных библиотек к проектам Windows.

Clean.props

Страница свойств, содержащая в качестве события перед построением удаление временных файлов, образующихся при построении проектов решения (должна подключаться к проекту Clean, который используется в скрипте полной сборки решения перед построением всех проектов решения).

RunSolutionTests.props

Страница свойств, содержащая в качестве события после построения скрипт запуска всех тестов решения в скрытом режиме и выводом результата в документацию решения (должен подключаться во все конфигурации x64 проекта BuildAll, используемого в скрипте построения всех проектов решения).

Coverage.props

Страница свойств, содержащая в качестве события после построения скрипт запуска всех тестов решения с подсчетом покрытия кода тестами (должна подключаться только к конфигурации Debug|x64 проекта BuildAll, используемого в скрипте построения всех проектов решения).

Должна идти в списке страниц свойств после RunSolutionTests.props.

Metrics.props

Вычисление метрик кода в решении

Example.props

Образец страницы свойств для копирования в папки внешних библиотек. Файл следует скопировать в папку библиотеки и добавить его как дочернюю страницу свойств в страницу свойств использующей его библиотеки. Содержит:

  • Макрос $(ThisFileDirectory): путь к папке, в которой расположен этот файл.
  • Дополнительный каталог включения заголовочных файлов $(ThisFileDirectory)/include.
  • Дополнительные каталоги поиска файлов библиотек $(ThisFileDirectory)/lib/$(PlatformTarget)/$(PlatformToolsetDirectory).
Заметки
Предполагается, что файлы библиотеки будут расположены в папке со следующей иерархией:
ExampleLib
|- ExampleLib.props
|- include // Папка заголовочных файлов
|- lib
...|- x86
...|..|- vs140 // Lib'ы x86 для VS2015
...|- x64
...|..|- vs140 // Lib'ы x64 для VS2015
...|- arm
......|- gcc_4_9 // Lib'ы ARM для GCC 4.9
