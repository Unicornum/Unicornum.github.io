Введение

Проект логики работы утилиты предварительной обработки решения перед построением документации.

Утилита Documentation

Проект утилиты предварительной обработки файлов решения.

Предназначен для:

  • Обработки файла лога сборки решения скриптом.
  • Обработки файла покрытия кода тестами.
  • Обработки лога сборки документации.
  • Установки переменных окружения, необходимых для построения документации.

Использование утилиты

  • Подключить к решению проект Documentation из папки Externals\Tools\Documentation.
  • Одному из проектов решения установить зависимость от проекта Documentation и в качестве события после построения добавить запуск Documentation.exe (в папке $(SolutionDir)$(Configuration)\Bin\x64) с нужными командами.

Параметры

  • –file=path - путь к исходному файлу (трактуется в зависимости от указанной команды).
  • –path-to-solution=path - полный путь к папке решения, которое обрабатывается (используется совместно с командами –process-report=build и –process-report=coverage).
  • –max-records - максимальное количество записей в отчете о покрытии кода тестами (при добавлении новых записей самые старые будут удаляться; используется совместно с командой –process-report=coverage).
  • –configuration - текущая собираемая платформа (используется совместно с командой –set-environment-variables).

Команды

Заметки
Команды исключают использование друг друга.
  • –help - выводит в консоль список доступных команд.
  • –process-report=type - обработка файла, заданного как параметр –file:
    • build - обработка файла лога сборки скриптом (генерируется файл, группирующий и подсвечивающий цветом ошибки и предупреждения по проектам).
    • coverage - обработка файла отчета покрытия кода тестами (из xml файла отчета генерируется html файлы, пригодные для включения в документацию Doxygen).
    • documentation - обработка файла лога сборки документации решения (исключаются вспомогательные сообщения, не несущие полезной информации).
  • –set-environment-variables - в консоль выводятся команды установки переменных окружения, устанавливающие значения версии решения:
    • DOXYGEN_NAME - название продукта.
    • DOXYGEN_VERSION - полная версия продукта.
    • DOXYGEN_BRIEF - краткое описание продукта.
    • DOXYGEN_INPUT - путь к папке, для которой следует построить документацию.
    • DOXYGEN_COPYRIGHT - строка copyright'a продукта.

Использование переменных окружения

Т.к. при установке переменных окружения внутри процесса они будут доступны только самому процессу, утилита выводит в консоль набор команд, которые можно использовать в командном файле следующим образом:

for /f "usebackq delims=;" %%a in (`documentation.exe --file=version.hpp --set-environment-variables`) do (
%%a
)
echo DOXYGEN_NAME: %DOXYGEN_NAME%
