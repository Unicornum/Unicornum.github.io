Testing

Введение

Раздел содержит программу-обертку над библиотекой gtest и комплект проектов тестирования. Назначение - удобная работа с отдельными классами и наглядное отображение результатов прогона тестов.

Подробное описание раздела

Комплект проектов тестирования содержится в папке Test.Front и включает в себя три проекта:

  • Test.Solution - единственный проект на все решение. Экспортируется в решение и предназначен для сборки и запуска всех тестов всех проектов решения. Собирается в папку \Result\Test\x64 и зависит ото всех проектов Test.Project. При запуске "пробегается" по всем dll-кам в своей папке, загружает их, выполняет расположенные в них тесты и отображает статистику выполнения тестов. При завершении возвращает 0, если все тесты прошли успешно и число > 0, если хотя бы один провалился.
  • Test.Project - отдельный тестовый проект для классов каждого проекта решения. Располагается в той же папке, в которой расположена папка проекта, тесты классов которого он будет содержать. 'Project' заменяется на имя того проекта, который будет тестироваться. Собирается как dll в папку \Result\Test\x86.
  • Test.Class - отдельный тестовый проект одного класса. Располагается в папке проекта Test.Project. Собирается как исполняемый файл для тестов одного класса. 'Class' в имени проекта заменяется на имя класса, тесты которого он будет исполнять.

Вся информация о прогоне тестов в проектах Test.Solution и Test.Class отображается в диалоговом окне, содержащем три закладки:

  • В первой закладке отображается суммарный результат (количество всех/пройденных/проваленных/отключенных тестов) в html-формате.
  • Во второй закладке отображается в виде дерева список всех тестов (разделы, содержащие проваленные тесты, помечаются красной иконкой). Иерархия должна содержать имя проекта теста, имя класса теста, имена тестов.
  • В третьей закладке отображаются только проваленные тесты. Выделив нужный тест, в нижней части окна можно получить подробное описание причины провала.

Развертывание

Подключение проектов тестирования возможно вручную (см. описание ниже) и автоматически - соответствующими командами надстройки UniCA.

Общий проект тестирования

  • Экспортировать проект Test.Solution в папку Solution решения и подключить к решению.
Заметки
  • Предназначен для запуска всех тестов решения.
  • Проект должен зависеть ото всех проектов Test.Project решения.

Проект тестирования классов одного проекта

  • В папку, в которой расположена папка проекта экпортировать папку 'Test.Project' и подключить его к решению (в раздел его проекта).
  • Заменить 'Project' на имя проекта, тесты классов которого он будет содержать.
  • Проекту Test.Solution добавить зависимость от добавленного проекта.
Заметки
В этот проект должны добавляться все файлы *_test.cpp всех проектов тестирования одного класса (в раздел Tests\ClassName).

Проект тестирования одного класса

  • Для каждого теста класса в папку проекта Test.Project экспортируется папка 'Test.Class'.
  • Заменить 'Class' на имя тестируемого класса.
  • Подключить добавленный проект к решению (в раздел Test.Classes проекта, к которому он относится).
  • Файл ClassName_test.cpp переименовать, заменив 'ClassName' на имя класса, после чего подключить его к проектам Test.Class и Test.Project.
  • В файле ClassName_test.cpp PathToHeader.hpp заменить на путь к файлу тестируемого класса.
  • В файле ClassName_test.cpp ClassName заменить на соответствующее имя класса.
