Подробное описание

Раздел классов реализации паттерна 'Фабрика'.

Примеры использования

Классы с конструкторами по умолчанию

Регистрация

using namespace ::alicorn::modules::patterns;
namespace example
{
class Type
{
public:
enum Value
{
Der1 = 0,
Der2,
TwoParams,
TwoParams2
};
};
// Интерфейсный класс, в указателях на который фабрика будет создавать объекты.
class IBase
{
protected:
using String_t = ::alicorn::extension::std::String;
public:
virtual String_t Test(void) const = 0;
virtual ~IBase() {}
};
// Реальный класс, объекты которого могут быть созданы через фабрику объектов.
class Der1 final :
public factory::Registrator<IBase>
{
public:
String_t Test(void) const override { return uT("Der1::Test()"); }
};
FACTORY_REGISTER(Der1, Type::Der1); // Регистрация по идентификатору.
// Реальный класс, объекты которого могут быть созданы через фабрику объектов.
class Der2 final :
public factory::Registrator<IBase>
{
public:
String_t Test(void) const override { return uT("Der2::Test()"); }
};
FACTORY_REGISTER_STRING_NAME(Der2); // Регистрация по строковому имени.
} // namespace example

Создание объектов в куче

using namespace ::alicorn::modules::patterns;
auto pObject = factory::make_unique<::example::IBase>(::example::Type::Der1);
ASSERT_TRUE(pObject != nullptr);
EXPECT_EQ(uT("Der1::Test()"), pObject->Test());
pObject = factory::make_unique<::example::IBase>(uT("Der2"));
ASSERT_TRUE(pObject != nullptr);
EXPECT_EQ(uT("Der2::Test()"), pObject->Test());

Создание объектов в пуле памяти

using namespace ::alicorn::modules::patterns;
auto pObject = factory::pool<>::make_unique<::example::IBase>(::example::Type::Der1);
ASSERT_TRUE(pObject != nullptr);
EXPECT_EQ(uT("Der1::Test()"), pObject->Test());
EXPECT_EQ(32 - 1, UsingPool.GetFreeObjects());
pObject = factory::pool<>::make_unique<::example::IBase>(uT("Der2"));
ASSERT_TRUE(pObject != nullptr);
EXPECT_EQ(uT("Der2::Test()"), pObject->Test());

Классы с конструкторами с параметрами

Регистрация

class ILValue
{
public:
virtual int GetIntValue(void) const = 0;
virtual ::std::string GetStrValue(void) const = 0;
public:
virtual ~ILValue() {}
};
class LValue :
public factory::Registrator<ILValue, const int &, const ::std::string &>
{
public:
int GetIntValue(void) const override { return m_iValue; }
::std::string GetStrValue(void) const override { return m_sValue; }
private:
const int m_iValue;
const ::std::string m_sValue;
public:
LValue(const int & _iValue, const ::std::string & _sValue) :
m_iValue(_iValue),
m_sValue(_sValue)
{
}
};
FACTORY_REGISTER(LValue, ::example::Type::TwoParams);

Создание объектов

using namespace ::alicorn::modules::patterns;
const int iValue = 1803122357;
const ::std::string sValue = "1803122358";
auto pExample = factory::make_unique<ILValue>(
::example::Type::TwoParams, iValue, sValue);
ASSERT_TRUE(pExample != nullptr);
EXPECT_EQ(iValue, pExample->GetIntValue());
EXPECT_EQ(sValue, pExample->GetStrValue());
pExample = factory::pool<>::make_unique<ILValue>(
::example::Type::TwoParams, iValue, sValue);
ASSERT_TRUE(pExample != nullptr);
EXPECT_EQ(iValue, pExample->GetIntValue());
EXPECT_EQ(sValue, pExample->GetStrValue());

Класс входит в Patterns
Макрос для регистрации класса в фабрике объектов.

  • При использовании этого макроса требуется унаследовать регистрируемый класс от Registrator<>.
  • Макрос ОБЯЗАТЕЛЬНО должен быть в заголовочном файле.
  • Если регистрируемый класс находится в пространстве имен, макрос также должен располагаться внутри этого пространства имен, а в качестве Class должно быть имя класса без его пространства имен.
Аргументы
[in]ClassРегистрируемый класс.
[in]ClassIdИдентификатор, по которому будут создаваться объекты класс (именно его следует передавать в функцию Factory<>::Make()).

◆ FACTORY_REGISTER_STRING_NAME

#define FACTORY_REGISTER_STRING_NAME (   Class)    FACTORY_REGISTER(Class, uT(#Class))

Класс входит в Patterns
Макрос для регистрации класса в фабрике объектов по строковому имени.

  • При использовании этого макроса требуется унаследовать регистрируемый класс от Registrator<>.
  • Класс будет зарегистрирован под строковым именем, совпадающим с именем класса.
  • Макрос ОБЯЗАТЕЛЬНО должен быть в заголовочном файле.
  • Если регистрируемый класс находится в пространстве имен, макрос также должен располагаться внутри этого пространства имен, а в качестве Class должно быть имя класса без его пространства имен.
Аргументы
[in]ClassРегистрируемый класс (регистрируется под собственным именем).
