◆ appdata_path()

path boost::filesystem::appdata_path ( void  )

Функция получения пути к папке данных программ текущего пользователя.

Android:

  • Для того, чтобы программа могла использовать эту папку, необходимо добавление в AndroidManifest.xml
    <manifest ...>
    ...
    <uses-permission android:name="android.permission.WRITE_EXTERNAL_STORAGE"/>
    <uses-permission android:name="android.permission.READ_EXTERNAL_STORAGE"/>
    </manifest>
Возвращает
Полный путь к папке данных программ текущего пользователя.
Исключения
std::exception
  • Действие невозможно (подробнее см. описание исключения).

◆ load_binary_file()

BinaryData_t boost::filesystem::load_binary_file ( const path &  _PathToFile,
size_t  _BeginPosition = 0,
size_t  _BytesCount = (size_t)-1 
)

Функция получения содержимого файла по указанному пути.

  • Если функции передать только путь к файлу, будет прочитан весь файл целиком.
  • Если начальная позиция превышает размер файла, функция вернет пустой набор данных.
  • Если (размер блока данных + начальная позиция) окажется за пределами фактического размера файла, функция вернет данные от начальной позиции до конца файла.

Android:

  • Если файл существует, функция вернет его содержимое.
  • Если файл не существует, функция попытается получить содержимое файла через AAssetManager (валидный путь - относительно папки assets, можно также использовать путь, возвращаемый CurrentModule::GetAppRootPath()).
Аргументы
[in]_PathToFileПуть к файлу, который будет прочитан.
[in]_BeginPositionНачальная позиция для чтения (от начала файла).
[in]_BytesCountРазмер читаемого блока данных в байтах.
Исключения
std::exceptionWindows:
  • Указанный файл не существует или недоступен для чтения.
    Android:
  • Указанный файл не существует физически и его нет в .apk файле внутри папки assets.
  • Указанный файл существует, но не доступен для чтения.
