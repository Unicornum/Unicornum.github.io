Подробное описание

Раздел классов работы с памятью.

Пример использования:

Захват ресурсов

using namespace ::alicorn::extension::std;
unique_ptr<FILE> Example{ fopen("file.txt", "r"), [](FILE * f) { fclose(f); } };

Создание объектов из пула памяти

using namespace ::alicorn::extension::std;
auto pExample = pool<4096>::make_unique<Example>();

Пример объявления класса с запретом создания объектов мимо пула памяти

class Component
{
using Pool_t = ::alicorn::extension::std::pool<>;
class ConstructorTag {}; // private!
public:
using ComponentPtr_t = ::std::shared_ptr<Component>;
public:
static ComponentPtr_t Make(void)
{
return Pool_t::make_unique<Component>(ConstructorTag{});
}
public:
Component(ConstructorTag)
{
// Параметр не используется. Предназначен для того, чтобы запретить
// создание объектов мимо функции Make(); закрытый конструктор использовать
// нельзя, т.к. в этом случае функция make_unique<>() также не сможет
// создавать объекты класса.
}
};

Преобразование

using namespace ::alicorn::extension::std;
auto pExample = make_unique<Test>(); // uinique_ptr<Test>
auto pIExample = make_shared(pExample); // ::std::shared_ptr<Test>
// Или, если нужен указатель на интерфейс...
// ::std::shared_ptr<ITest> pIExample = make_shared(pExample);
// Здесь pExample уже равен nullptr

◆ unique_ptr

template<class T >
using alicorn::extension::std::unique_ptr = typedef ::std::unique_ptr<T, void(*)(void *)>

Класс входит в проект Std
Вспомогательное определение аналога std::unique_ptr<> для захвата ресурсов.

  • В качестве deleter'a можно задавать лямбда-выражение (std::function<void(T *)> не используется, т.к. в этом случае вызывается его оператор присваивания при создании объекта unique_ptr<>, что существенно сказывается на времени создания объекта из пула).
  • Если при создании объекта будет возвращен nullptr, deleter вызван не будет.

TКласс, в указателе на который будут храниться созданные объекты.
