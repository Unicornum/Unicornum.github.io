Класс входит в проект Std
Класс создания объектов из пула памяти.

  • Класс потокобезопасен (в каждом потоке будет свой объект пула, поэтому уничтожать объекты следует в том же потоке, в котором он был создан, а функцию Release() следует вызывать отдельно для каждого потока).
  • Создание объектов из пула памяти не приводит к фрагментации кучи.
  • Создание объектов из пула памяти происходит в 5+ раз быстрее, чем оператором new.
  • Использовать класс напрямую не рекомендуется, для создания объектов из пула следует использовать функцию pool<>::make_unique<>().

inline

Функция создания объекта из пула.

  • Функция возвращает полноценные указатели на объекты (для которых, например, будет работать dynamic_cast<>()).
Аргументы
[in]_BlockSizeРазмер блока памяти, который будет выделяться за раз (при создании первого объекта будет запрошена память в объеме _BlockSize * sizeof(T), в дальнейшем по мере использования этого объема, память будет выделяться блоками такого же размера).
[in]_ArgsПараметры, передаваемые конструктору создаваемого объекта.
Исключения
std::exception
  • Действие невозможно (подробнее см. описание исключения).

◆ Release()

template<class T >
void alicorn::extension::std::memory::Pool< T >::Release ( void )
inline

Функция удаления всей выделенной для пула памяти.

  • Все созданные из пула объекты после вызова этой функции станут невалидными, поэтому следует гарантировать разрушение всех объектов до вызова функции.
  • Деструктора объектов, которые не были освобождены вызовом функции Deallocate() вызваны не будут.
