CurrentModule::Path_t alicorn::system::application::CurrentModule::GetAppDataPath ( void )
static

Функция получения пути к папке, содержащей рабочие данные программы.

Возвращает
Android
  • Путь к папке, в которой программа может сохранять промежуточные данные.
  • Данные по этому пути будут сохранены даже при удалении программы с компьютера.
  • Папка, путь к которой возвращает эта функция гарантированно существует.
  • Данные в этой папке программа может менять.
  • Если такого пути не существует, он будет создан.
Windows
  • Для каждого пользователя это будет свой уникальный путь.
  • Формируется из пути к папке данных программ текущего пользователя, названий компании и продукта.
  • Названия компании и продукта будут извлечены из версии текущего модуля (будут совпадать с макросами __COMPANY_NAME__ и __PRODUCT_NAME__ текущего решения), поэтому у программы, желающей использовать эту возможность должен быть подключен файл с ресурсом версии программы.
  • Если у текущего модуля нет ресурса версии, функция вернет путь к корневой папке расположения модуля (см. GetAppRootPath()).

◆ GetAppRootPath()

CurrentModule::Path_t alicorn::system::application::CurrentModule::GetAppRootPath ( void )
static

Функция получения пути к корневой папке расположения программы.

Возвращает
Android
  • Этот путь можно использовать для формирования пути к файлам программы, расположенным в папке assets.
  • Данные в этой папке можно только читать, изменять их нельзя.
Windows
  • Если путь к модулю включает в себя папку Bin (или bin), функция вернет путь к ее родительской папке, в противном случае - путь к папке расположения модуля (см. GetLocation()).
Исключения
std::exception
  • Действие невозможно (подробнее см. описание исключения).
