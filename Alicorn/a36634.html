Использование утилит

Введение

Раздел проектов вспомогательных утилит.

Подробное описание

  • Проекты этого раздела, которые строят исполняемые файлы, содержатся в Externals и при необходимости использования должны подключаться к решению, чтобы содержать правильное содержимое в ресурсе версии.
  • Для возможности подключения этих проектов необходимо, чтобы в папке расположения файла страницы свойств Solution.x64.props должен находиться файл пустой страницы свойств Tools.props (это плата за возможность использования каскадных ссылок на разделы внутири Alicorn).

Проекты раздела

Проект Settings

Проект программы редактирования настроек.

  • При запуске программа открывает настройки решения, в рамках которого была собрана и позволяет изменять значения параметров.

Проект Version

Проект утилиты формирования файла с версией решения, предназначенный для включения в проект Packaging Android программы.

  • Один из проектов, от которого зависит Packaging, должен в качестве события после построения запускать утилиту, передавая ей в качестве параметра командной строки путь к папке, в которой расположен файл AndroidManifest.xml проекта.
  • В файл AndroidManifest.xml утилита будет вписывать текущую ревизию решения:
    <manifest ...
    android:versionCode="CURRENT_REVISION">
    ...
  • Во вложенной папке res/values утилита будет формировать файл version.xml, содержащий параметры версии решения (его необходимо включить в проект Packaging). Пример формируемого файла со строками версии:
    <?xml version="1.0" encoding="utf-8"?>
    <resources>
    <string name="app_version">1.2.3.2015</string>
    <string name="app_name">Название программы (debug)</string>
    <integer name="app_revision">456</integer>
    </resources>
  • Строки версий решения можно использовать следующим образом - в файле AndroidManifest.xml изменить:
    <manifest ...
    android:versionName="@string/app_version"
    ...

Проект Localization

Проект утилиты генерации заголовочного файла с идетификаторами строк, используемыми для получения у модуля локализации программы текста выбранного языка программы.

  • Утилита генерирует заголовочный файл из xml файла, переданного в качестве параметра.
  • Значение узла path будет интерпретироваться как путь к папке расположения файлов локализации программы (относительно корневой папки расположения exe файла программы).
  • Узлы, не содержащие значений будут интерпретироваться как указание создать пространство имен с именем узла.
  • Узлы, содержащие значения, будут интерпретироваться как строки, которые будет использовать программа (для них будут сгенерированы идентификаторы).
  • При обработке переданного файла утилита также будет сравнивать с указанным файлом все xml файлы в той же папке с выводом найденных расхождений в консоль.

Использование утилиты

Проект Resource

Проект утилиты генерации заголовочного файла, содержащего бинарные данные указанного файла. Назначение - хранение непосредственно в программе ресурсов небольшого размера (иконок, курсоров и т.п.).

  • Если в качестве пути (path="path/to/...") задан путь к файлу, заголовочный файл генерируется только для него, если путь к папке - то для всех файлов в этой папке, исключая файлы .hpp.
  • Сгенерированный заголовочный файл будет располагаться в той же папке, что и исходный файл и будет иметь то же имя (с расширением .hpp).
  • Имя объекта в созданном файле будет совпадать с именем исходного файла.
