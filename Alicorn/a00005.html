Раздел Externals

Введение

Раздел для внешних библиотек.

Заметки
Папка Externals игнорируется Doxygen при обработке, поэтому файл описания раздела находится в папке документации решения.

Файлы свойств

Solution.props

Общая страница свойств решения.

  • Все проекты должны включать эту страницу свойств (это обеспечивается надстройкой Properties).
  • Рекомендуемое расположение - в папке '\Solution'.
  • Содержит свойства, специфичные для данного решения.
  • Должна включать как дочернюю страницу свойств Externals\Common.props.

Common.props

Страница свойств, содержащая общие свойства:

  • Макрос $(IsBinary): 'Yes' для проектов Exe и Dll и 'No' для Lib.
  • Макрос $(IsTest): 'Yes' для проектов тестирования.
  • Макрос $(PartName): имя раздела проекта (состоит из имен папок между папкой решения и проекта, разделенных точками).
  • Макрос $(TypeProjectDir): имя вложенной папки для результата построения проекта ('Bin' для Exe и Dll, 'Lib' для Lib и 'Test' для проектов тестирования).
  • Путь к выходному каталогу (папка Result в папке решения + тип проекта (Bin, Lib или Test) + имя платформы (x86 или x64)).
  • Путь к промежуточному каталогу (в папку Temp в папке решения + промежуточные папки, чтобы исключить конфликт проектов с одинаковыми именами в разных разделах).
  • Имя файла результата (к проекту статической библиотеки добавляется имя раздела и имя конфигурации).
  • Путь '$(SolutionDir)\Solution\Solution' в качестве дополнительного каталога включения заголовочных файлов.
  • Формат отладочной информации установлен в 'База данных программы', чтобы убрать предупреждение 'warning D9007: для "/Gm" требуется "/Zi или /ZI"'.
  • Включен 4-й уровень предупреждений (в создаваемых проектах наследовать свойство от родителя явным образом, т.к. по умолчанию установлен 3-й уровень предупреждений).
  • Параметр 'Компоновать библиотеки из зависимостей' установлен в 'Нет' для того, чтобы exe-шники, зависящие от dll не требовали соответствующих .lib файлов.
  • Создание .pdb файла в папке промежуточного результата.

Lib.props

Образец страницы свойств для копирования в папки внешних библиотек. Файл следует скопировать в папку библиотеки и добавить его как дочернюю страницу свойств для страницы свойств Common.props. Содержит:

  • Макрос : путь к папке, в которой расположен этот файл.
  • Дополнительный каталог включения заголовочных файлов \include.
  • Дополнительные каталоги поиска файлов библиотек \Lib\ и \Lib\\vs.
Заметки
Предполагается, что файлы библиотеки будут расположены в папке со следующей иерархией:
ExampleLib
|- include // Папка заголовочных файлов
|- lib
...|- x86 // Lib'ы x86 для всех версий Visual Studio
...|..|- vs120 // Lib'ы x86 только для VS2013
...|- x64 // Lib'ы x64 для всех версий Visual Studio
......|- vs120 // Lib'ы x64 только для VS2013

Библиотеки

Boost

Домашняя страница: Boost.

Построение всех возможных вариантов:

bootstrap.bat
b2.exe

Построение только нужных библиотек:

bootstrap.bat
b2.exe --build-type=minimal msvc stage --with-filesystem

Построение для конкретной версии Visual Studio:

// Для VS2008
bootstrap.bat vc9
b2.exe toolset=msvc-9.0

Построение для x64.

Конфликт в DLL, построенной на базе MFC

Проявляется как assert'ы в файлах dllinit.cpp и dllmodul.cpp при загрузке DLL.

Для решения проблемы, закомментировать в файле ./libs/thread/src/win32/tss_pe.cpp строку

// extern BOOL (WINAPI * const _pRawDllMain)(HANDLE, DWORD, LPVOID)=&dll_callback;

и пересобрать lib'ы. Также нужно будет при выгрузке библиотеки вызвать

#include <boost/thread/detail/tss_hooks.hpp>
on_process_exit();

Poco

Домашняя страница Poco.

Построение

Использование buildwin.cmd:

buildwin.cmd
Usage:
------
buildwin VS_VERSION [ACTION] [LINKMODE] [CONFIGURATION] [PLATFORM] [SAMPLES] [TESTS] [TOOL]
VS_VERSION: "90|100|110|120"
ACTION: "build|rebuild|clean"
LINKMODE: "static_mt|static_md|shared|all"
CONFIGURATION: "release|debug|both"
PLATFORM: "Win32|x64|WinCE|WEC2013"
SAMPLES: "samples|nosamples"
TESTS: "tests|notests"
TOOL: "devenv|vcexpress|wdexpress|msbuild"
Default is build all.

Для построения без отладочной информации открыть нужное решение в папке Foundation, в свойствах проекта в разделе "C\C++" параметр "Формат отладочных данных" установить в "Нет".

wxWidgets

Домашняя страница: wxWidgets.

Построение для Windows

  • Скачать инсталлятор.
  • Установить библиотеку.
  • Открыть решение для нужной версии Visual Studio в папке build\msw.
  • Отключить UniCA (чтобы она не подключала Version.rc) и собрать все решение (результат построения будет в папке lib).
  • Не используемые библиотеки добавить в Lib.props (параметр "Игнорировать библиотеки в разделе Компоновщик -> Ввод).
