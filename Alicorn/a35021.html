Logger

Список классов проекта: Logger

Введение

Библиотека логирования программы предназначена для ведения журнала работы программы.
Запись сообщений в журнал производится путем добавления в код программы записей вида:

// Записать в лог сообщение "Найдено 5 элементов"
LOGGER(Warning) << uT("Найдено ") << 5 << " элементов";
Предусловие
Предполагается, что в настройках проекта, использующего логгер включена подстановка полного пути к файлу для макроса _ _ FILE _ _.

Подробное описание

  1. Простое добавление в сообщения журнала значений переменных произвольных типов (поддерживаются все типы, для которых определен оператор << (std::wostream &)).
  2. Несколько типов сообщений с различным приоритетом.
  3. Выполнение дополнительного действия для каждого типа сообщения.
  4. Формирование журнала событий в различных форматах.
  5. "Роллинг" файлов журнала (хранится заданное число журналов, при создании нового самый старый удаляется).
  6. Задание ограничения размера файла журнала (при превышении заданного значения текущий файл журнала будет закрыт и создан новый).
  7. Фильтрация выводимых сообщений по заданному приоритету.
  8. Настройка работы логгера в программе или файле настроек программы.
  9. В случае совпадения имени файла создаваемого журнала с уже существующим к имени файла будет добавлен порядковый номер в виде "_xxxx".
  10. Логгер является потокобезопасным для всех выполняемых им действий.

Используемые типы сообщений

По убыванию приоритета:

  • Fatal - фатальная ошибка (после записи сообщения в журнал будет брошено исключение alicorn::extension::std::exception::Abort).
  • Error - серьезная ошибка (нет дополнительного действия).
  • Warning - предупреждение (нет дополнительного действия).
  • Info - информационное сообщение (нет дополнительного действия).
  • Trace - сообщение (нет дополнительного действия).

В выходной лог записывается:

  • Имя и полная версия запущенной программы (один раз в заголовок лога).
  • Дата/время (с микросекундами) записи события.
  • Идентификатор потока, из которого производилась запись.
  • Тип события.
  • Сообщение, установленное для события.
  • Место (имя файла/номер строки) записи события (полный путь к файлу будет урезан до папки расположения файла).

Форматы ведения журнала:

  • Вывод сообщений в консоль.
  • Текстовый файл.
  • HTML файл (подЦВЕТка разных типов сообщений).
  • Окно "Вывод" Visual Studio (формат вывода сделан таким, чтобы по двойному клику на сообщении можно было сразу перейти к месту, где это сообщение было записано).

Подключение

Для использованя логгера в файл stdafx.h добавить строку

#include <alicorn\logger.hpp>

До первой записи в лог следует установить формат ведения журнала:

// Вывод в текстовый файл.
LOGGER(report::Text);

Использование

Запись сообщения в лог производится при помощи макросов LOGGER(), LOGGER_DEBUG() (только в отладочной версии программы) и оператора <<.
В цепочке операторов << может быть сколько угодно значений (причем могут быть использованы любые типы, для которых определен оператор << вывода в wostream), в качестве сообщения в журнал будет записана вся цепочка до точки с запятой.

Предупреждения
Поскольку логгер для работы использует статические объекты, запись в лог нельзя производить в деструкторах статических объектов.

Конфигурирование

Установка параметров логирования производится в следующем порядке:

  1. Настройки по умолчанию.
  2. Установка параметров в программе (это как бы настройки по умолчанию для этой программы).
  3. Загрузка параметров из файла настроек программы (подробности см. Settings).

Параметры по умолчанию

  • Имя программы - берется из ресурса версии ('ProductName').
  • Версия программы - берется из ресурса версии ('ProductVersion').
  • Папка журналов - папка с именем Logs в папке данных программы (см. alicorn::system::CurrentModule::GetAppDataPath()).
  • Количество файлов журнала не ограничено.
  • Размер файла журнала не ограничен.
  • Выводятся сообщения Warning и выше.
  • Разрешена повторная запись сообщений из одного и того же места (файла/строки).
  • Тип вывода журнала не задан.

Конфигурирование в программе

Настройка логгера в программе производится при помощи макроса LOGGER(); в качестве параметра ему передается тип задаемого параметра, а значение - при помощи оператора << и макроса LOGGER_MIN_LEVEL(), например:

// Использовать вывод в простом текстовом формате.
LOGGER(report::Text);
// Выводить в лог только ошибки типов Fatal и Error.
LOGGER(Filter) << LOGGER_MIN_LEVEL(Error);
// Хранить 10 последних журналов.
LOGGER(MaxLogFiles) << 10;
Заметки
  • Изменение настроек логирования может производиться в любой момент работы программы, однако значения, прочитанные из файла настроек программы перезаписаны не будут.
  • При установке нового типа журнала предыдущий будет удален.

Создание собственного типа журнала

  • Создать класс, унаследованный от класса alicorn::logger::report::Report и определить для него функции интерфейса alicorn::logger::report::IReport.
  • Разместить его впространстве имен alicorn::logger::report.

Конфигурирование в файле настроек программы

Пример задания настроек в разделе Logger в файле Settings.xml

<?xml version="1.0" encoding="utf-8"?>
<root>
<settings>
<Logger parent="">
<PathToLogDirectory>Logs</PathToLogDirectory>
<MaxLogFiles>10</MaxLogFiles>
<MaxLogSize>10000000</MaxLogSize>
<Filter>0</Filter>
<AllowReplace>true</AllowReplace>
</Logger>
</settings>
</root>
