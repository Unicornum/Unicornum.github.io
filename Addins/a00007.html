Исходные требования

Введение

Продукт предназначен для автоматизации рутинных операций при работе с кодом и упрощения работы со вспомогательными утилитами (прямо из среды разработки).

Общее описание

Продукт представляет собой набор надстроек для IDE, а также вспомогательные утилиты, которые могут быть запущены из IDE (выбором строки меню).

Общие требования:

  • Полезная информация о работе продукта, ошибки и предупреждения должны выводиться в окно "Вывод" IDE, с открытием этого окна при появлении в нем новой записи об ошибке/предупреждении.

Функциональность

Настройка продукта

Настройки продукта располагаются в файле settings.xml в папке продукта (настройки каждого модуля - в своем разделе; для каждой версии IDE также отводится свой раздел). Изменение настроек производится при помощи программы settings.exe в папке продукта или выбором соответствующего пункта меню.

Раздел Macros

Раздел команд в общем меню для ввода типичных кусков кода и назначающая им "горячие" клавиши:

  • Добавление строки "#pragma chMSG()" (ALT + M).
  • Перевод строки в комментарии (ALT + 8).
  • Добавление заготовки текста _T("") (ALT + T).
  • Добавление строки даты/времени (в виде 1602061200) (ALT + D).

CppCheck

Проверка кода статическим анализатором CppCheck.

Общие требования

  • Шаблон выводных сообщений: '{file}({line}): {severity} (suppress_id = {id}): {message}'.
  • Включены все сообщения (–enable=all).
  • Игнорирование помеченных специальным комментарием строк (–inline-suppr).
  • Показывать подозрительные места (–inconclusive).
  • Игнорировать папку внешних библиотек.
  • В разных версиях Visual Studio должны производиться проверки с учетом соответствующих стандартов (–std=c++03 для VS2008 или –std=c++11 для VS2013).

Игнорировать строки, содержащие:

  • 'information (suppress_id = purgedConfiguration):'.

Быстрая проверка

  • Запуск CppCheck с заданием в качестве путей include только пути "<SolutionPath>\Solution\Solution".
  • Проверка со всеми вариантами макросов (–force).

Version

Надстройка должна обеспечивать:

  • Все .exe и .dll файлы при сборке должны содержать версию исходных файлов + ревизию файлов решения.
  • Макрос, представляющий собой строку текущей версии программы.
  • Формат версии '1.0.0.1234 Build ЧЧ:ММ ДД.ММ.ГГГГ'.

Properties

Надстройка, подключающая всем проектам решения общий файл .props.

  • Путь к общему файлу свойств берется из настроек.
  • При открытии решения надстройка должна для всех проектов проверять наличие общего файла свойств.
  • При отсутствии файла свойств у проекта он должен подключаться.
  • При удалении проекта из решения, общий файл свойств должен отключаться от проекта.

Прототип: https://cacoo.com/diagrams/4AYESADorTeoeyKf

Порядок работы с программой

  • При запуске программы отображается вкладка Match, на ней активна вкладка 'Тестовые строки'.
  • В поле регулярного выражения введено простое регулярное выражение для примера.
  • В поле тестовых строк выведены строки - подсказки ('переключитесь на вкладку результата, чтобы увидеть результат обработки'), совпадающие и не совпадающие с регулярным выражением.
  • Результат обработки отображается в html-формате.
  • При вводе символов в поле регулярного выражения программа автоматически переключается на вкладку результата обработки.
  • При переключении вкладок в соответствующие поля копируются регулярное выражение и тестовые строки.
  • Вкладки Match и Search идентичны, во вкладке Replace поля ввода регулярного выражения и строки на замену находятся на разных вкладках вложенного tab'a.
  • При переключении вкладок обновлять представление и переключаться на вкладку исходных строк.
  • При клике правой кнопкой мыши в окне регулярного выражения появляется выпадающее меню, которое позволяет скопировать/вставить регулярное выражение в режиме 'как есть' и 'для вставки в код С++'.
  • Регулярное выражение должно отображаться крупно, полужирным шрифтом, а пробелы должны заменяться символом U+00B7.
  • При выделении тестовой строки показывать результат обработки только этой строки?
