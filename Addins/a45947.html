Требования модуля Time Tracker
  • Ключевое требование! Все действия должны происходить автоматически, пользователь вообще не должен участвовать в процессе учета времени.
  • Затраченное время учитывается по действиям пользователя во время работы с Visual Studio (изменение текста открытого документа, выбор другого элемента в обозревателе решений, переключение активного окна).
  • Учтенное время работы должно записываться на специальную страницу документации в виде отчета в формате Doxygen.
  • Команда вызова окна просмотра отчета за две недели (для текущего решения и суммарный для всех решений, в которые вносились исправления).

Учет времени

  • Промежутки веремени между событиями более (параметр IgnoreTime в разделе DevelopmentInfo настроек) не должны учитываться.
  • Запись времени работы в файлы через промежутки времени, заданные параметром TimeSaveChange в разделе DevelopmentInfo настроек.

Страница информации о времени работы с решением

  • Содержит информацию о общем времени, потраченном на разработку решения.
  • Содержит сгенерированный отчет для Doxygen.

Страница информации обо всех решениях, измененных за последние 2 недели

  • Содержит статистику времени работы с решениями за последние две недели (раздельно по дням недели).

Отчет за неделю

  • Отдельная утилита, получающая в качестве параметра пути к файлам, содержащим статистику работы с текущим решением и со всеми решениями, измененными за последние две недели.
