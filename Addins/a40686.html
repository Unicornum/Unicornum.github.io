Набор инструментов, упрощающих работу над кодом.
Исходные требования

Общие требования:

  • Полезная информация о работе продукта, ошибки и предупреждения должны выводиться в окно "Вывод" IDE, с открытием этого окна при появлении в нем новой записи об ошибке/предупреждении.

Команды

Раздел Macros

Раздел команд в общем меню для ввода типичных кусков кода и назначающая им "горячие" клавиши:

  • Добавление строки "TODO()" (ALT + M).
  • Перевод строки в комментарии (ALT + 8).
  • Добавление заготовки текста _T("") (ALT + T).
  • Добавление заготовки текста L"" (ALT + W).
  • Добавление заготовки текста uT("") (ALT + U).
  • Добавление строки даты/времени (в виде 1602061200) (ALT + D).

CppCheck

Проверка кода статическим анализатором CppCheck.

Общие требования

  • Шаблон выводных сообщений: '{file}({line}): {severity} (suppress_id = {id}): {message}'.
  • Включены все сообщения (–enable=all).
  • Игнорирование помеченных специальным комментарием строк (–inline-suppr).
  • Показывать подозрительные места (–inconclusive).
  • Игнорировать папку внешних библиотек.
  • В разных версиях Visual Studio должны производиться проверки с учетом соответствующих стандартов (–std=c++03 для VS2008 или –std=c++11 для VS2013+).

Игнорировать строки, содержащие:

  • 'information (suppress_id = purgedConfiguration):'.

Быстрая проверка

  • Запуск CppCheck с заданием в качестве путей include только пути "<SolutionPath>\Solution\Solution".
  • Проверка со всеми вариантами макросов (–force).

Модули

Version

Надстройка должна обеспечивать:

  • Все .exe и .dll файлы при сборке должны содержать версию исходных файлов + ревизию файлов решения.
  • Макрос, представляющий собой строку текущей версии программы.
  • Формат версии '1.0.0.1234 Build ЧЧ:ММ ДД.ММ.ГГГГ'.

Properties

Надстройка, подключающая всем проектам решения общий файл .props.

  • Путь к общему файлу свойств берется из настроек.
  • При открытии решения надстройка должна для всех проектов проверять наличие общего файла свойств.
  • При отсутствии файла свойств у проекта он должен подключаться.
  • При удалении проекта из решения, общий файл свойств должен отключаться от проекта.

Time Tracker

  • Ключевое требование! Все действия должны происходить автоматически, пользователь вообще не должен участвовать в процессе учета времени.
  • Затраченное время учитывается по действиям пользователя во время работы с Visual Studio (изменение текста открытого документа, выбор другого элемента в обозревателе решений, переключение активного окна).
  • Учтенное время работы должно записываться на специальную страницу документации в виде отчета в формате Doxygen.
  • Команда вызова окна просмотра отчета за две недели (для текущего решения и суммарный для всех решений, в которые вносились исправления).

Учет времени

  • Промежутки веремени между событиями более (параметр IgnoreTime в разделе DevelopmentInfo настроек) не должны учитываться.
  • Запись времени работы в файлы через промежутки времени, заданные параметром TimeSaveChange в разделе DevelopmentInfo настроек.

Страница информации о времени работы с решением

  • Содержит информацию о общем времени, потраченном на разработку решения.
  • Содержит сгенерированный отчет для Doxygen.

Страница информации обо всех решениях, измененных за последние 2 недели

  • Содержит статистику времени работы с решениями за последние две недели (раздельно по дням недели).

Отчет за неделю

  • Отдельная утилита, получающая в качестве параметра пути к файлам, содержащим статистику работы с текущим решением и со всеми решениями, измененными за последние две недели.
