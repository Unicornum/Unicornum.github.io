Исходные требования утилиты ProjectTasks

Общее описание

Исходные данные

В качестве параметров командной строки программа должна получать:

  • Имя проекта (с путем внутри решения).
  • Путь к файлу, содержащему список задач проекта.
  • Путь к файлу истории решения.
  • Номер дня, с которого начинается неделя.
  • Количество недель, для которого будет храниться список завершенных задач.

Настройки списка задач

  • Номер дня, с которого начинается неделя
  • Путь к файлу списка задач относительно родительской папки проекта.
  • Путь к файлу истории решения.
  • Количество недель, для которых следует хранить завершенные задачи.

Требования первого этапа

  • Команда выпадающего меню проекта (UniCA) открытия окна, отображающего список задач.

Главное диалоговое окно программы

  • Отображение в заголовке окна имени проекта, для которого отображается список задач.
  • Вкладка со списком текущих задач.
  • Вкладка со списком завершенных задач.
  • Вкладка 'О программе'.
  • Все изменения должны записываться в исходный файл списка задач при закрытии окна.

Вкладка со списком задач

  • Задачи должны отображаться в виде иерархического дерева, позволяющего создавать подзадачи.
  • Корневой раздел 'В процессе', содержащий список задач, находящихся в работе.
  • Перетаскивание задач мышью из одного узла в другой (вместе с дочерними узлами).
  • Задача представляет собой строку, содержащую текст задачи и набор кнопок для управления задачей.
  • Кнопка 'Редактировать' позволяет изменить текст задачи.
  • Кнопка 'Создать' позволяет создать дочернюю задачу для текущей.
  • CheckBox 'Завершить' позволяет завершить задачу.
  • При завершении задачи из корневого раздела, она переноситься во вкладку завершенных задач, в раздел текущей недели.

Вкладка со списком завершенных задач

  • Содержит разделы задач, завершенных за несколько последних недель.
  • Запрещена манипуляция задачами (изменение порядка расположения, перенос задачи из одной родительской задачи в другую, редактирование параметров задачи).
  • Задача представляет собой строку, содержащую текст задачи и набор кнопок для управления задачей.
  • Дочерняя задача раздела недели должна быть помечена как выполненная с возможность ее 'развыполнения' (с переносом в родительскую задачу списка текущих задач).

Требования второго этапа

  • При завершении задачи добавляется запись в файл истории решения, содержащая дату/время завершения, имя проекта к которому она относилась и текст задачи.
  • Ротация файлов истории изменений решения при превышении заданного в настройках размера.

Возможность завершения задачи

  • Признак того, что задача может/не может быть завершена.
  • Задача, имеющая незавершенные дочерние задачи должна быть помечена как запрещенная к завершению.
  • При завершении дочерней задачи, она переносится в конец списка родительской задачи с пометкой 'завершенная'.
  • При завершении задачи все ее дочерние задачи должны быть помечены как запрещенные к завершению.

Тип задачи

  • Отображается иконкой перед описанием задачи, выбирается из выпадающего списка на панели редактирования задачи.
  • Типы в порядке убывания приоритета: Баг, Важное срочное, Важное несрочное, Неважное несрочное, Для информации, Нет.

Действия при завершении задачи

  • Выпадающий список: Ничего/Увеличить номер версии/подверсии/подподверсии.
  • Выполнить действие, заданное в настройках для папки проекта/раздела/решения.
  • Параметр: путь к программе, выполняемой при завершении задачи.
  • Параметр: параметры командной строки, содержащие макросы, которые будут заменяться путем к проекту/разделу/решению и описанием завершаемой задачи.
