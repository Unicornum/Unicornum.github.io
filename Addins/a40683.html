Вставить...

Команды вставляют в активный документ (в то место, где находится курсор) указанные заготовки текста. Строка даты/времени - строка вида '1805241232', которая является уникальной при каждом выполнении команды, предназначена для использования при создании тестовых данных.

Перенос строки комментария - завершение текущей строки с добавлением в начало новой строки префикса комментария, совпадающего с префиксом текущей строки ('// ', '/// ' или '* ').

Генерировать документацию решения

Команда запускает генерацию документации решения при помощи Doxygen, после чего открывает собранную документацию в браузере по умолчанию.

Предупреждения Doxygen при обработке выводятся в выводное окно Visual Studio в раздел Doxygen.

Перед запуском программы построения документации производятся дополнительные действия:

  • Обновляется ревизия файлов решения в файле версии решения.
  • Обрабатывается файл лога сборки решения скриптом (txt -> html) для группировки сообщений по проектам и подсветки ошибок и предупреждений цветом.
  • Обрабатывается файл отчета об измерении покрытия кода тестами и генерируюся файлы в html формате для включения в документацию решения.

Используемые параметры настроек

Описания параметров см. в утилите редактирования настроек.

  • Documentation -> PathToCoverageResult.
  • Documentation -> PathToDoxygenBin.
  • Documentation -> PathToDoxygenIniFile.
  • Documentation -> PathToMSBuildResult.

CppCheck: быстрая проверка решения

Команда запускает быструю (без учета заголовочных файлов используемых сторонних библиотек) проверку кода в решении.

Лог обработки выводится в выводное окно Visual Studio в раздел CppCheck.

Для исключения ложных срабатываний можно использовать файл, путь к которому задается в настройках.
Формат записей: '[error id]:[filename]:[line]'.

Используемые параметры настроек

Описания параметров см. в утилите редактирования настроек.

  • Utilities -> PathToCppCheckBin.
  • Utilities -> PathToCppCheckSuppressFile.

Генерировать подставной класс

Команда генерирует подставной класс для класса, который объявлен в активном документе:

  • Имя заголовочного файла подставного класса будет совпадать с именем файла активного документа с добавлением суффикса .mock.
  • Сгенерированный файл будет располагаться в папке файла активного документа.
  • Подставной класс будет иметь тоже имя, что и исходный и расположен он будет в том же пространстве имен, что и исходный класс, но 'обернутый' в пространство имен mock.
  • Все открытые функции подставного класса будут заменены объявлениями подставных функций для использования библиотеки Google Mock.
  • Подставной класс будет сгенерирован только если файл поставного класса не существует.
  • Сгенерированный заголовочный файл подставного класса будет включен в общий заголовочный файл подставных классов проекта.

Генерировать подставной класс (+ прокси)

Команда генерирует подставной класс для класса, который объявлен в активном документе:

  • Имя заголовочного файла подставного класса будет совпадать с именем файла активного документа с добавлением суффикса .mock.
  • Сгенерированный файл будет располагаться в папке файла активного документа.
  • Подставной класс будет иметь тоже имя, что и исходный и расположен он будет в том же пространстве имен, что и исходный класс, но 'обернутый' в пространство имен mock.
  • Все открытые функции подставного класса будут вызывать одноименные функции специального вложенного прокси-класса, которые объявлены как подставные функций для использования библиотеки Google Mock.
  • Подставной класс будет сгенерирован только если файл поставного класса не существует.
  • Сгенерированный заголовочный файл подставного класса будет включен в общий заголовочный файл подставных классов проекта.
Заметки
Использование подставного класса с промежуточным прокси-классом позволяет проверять вызовы конструкторов, деструкторов и статических функций.

Генерировать подставной класс (inline)

Команда генерирует файл с подставными inline реализациями функций класса, который объявлен в активном документе:

  • Имя заголовочного файла подставного класса будет совпадать с именем файла активного документа с добавлением суффикса .mock.
  • Сгенерированный файл будет располагаться в папке файла активного документа.
  • Прокси-класс будет иметь тоже имя, что и исходный и расположен он будет в том же пространстве имен, что и исходный класс, но в дополнительном пространстве имен mock.
  • Все реализации открытых функций будут вызывать одноименные функции специального прокси-класса, которые объявлены как подставные функций для использования библиотеки Google Mock.
  • Подставная реализация будет сгенерирована только если файл поставного класса не существует.
  • Сгенерированный заголовочный файл подставной реализации будет включен в общий заголовочный файл подставных классов проекта.
Заметки
  • Использование подставного класса с промежуточным прокси-классом позволяет проверять вызовы конструкторов, деструкторов и статических функций.
  • Для использования таким образом созданного подставного класса не требуется манипуляция пространством имен.

Показать различия/Убрать изменения для файла документа

Команда вызывает соответствующее окно TortoiseSVN для файла активного документа.

Построить диаграмму UML для файла документа

Команда создает диаграмму UML для файла активного документа:

  • Используется запрос к серверу PlantUML, поэтому для работы команды необходим доступ в Интернет, синтаксис документа для построения диаграмм можно посмотреть там же.
  • Файл диаграммы будет создан в той же папке, что и файл активного документа, с тем же именем, но расширением .png.
  • Исходный файл диаграммы должен быть в формате Utf8 с сигнатурой.

Команда добавляет выделенный текст в специальный файл, который Visual Studio использует как источник слов, которые будут подсвечиваться в редакторе.
