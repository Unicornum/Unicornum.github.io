Подробное описание

Класс входит в проект Covellite.Core
Базовый класс приложения для Android.

  • Содержит главный цикл приложения и вызов сигналов для полученных сообщений.
  • Для создания объекта приложения следует создать класс-наследник от этого класса и в его конструкторе подписаться на сообщения, которые следует обрабатывать.
  • Для того, чтобы при смене ориентации устройства программа не завершалась, необходимо в AndroidManifest.xml добавить
    <activity ...
    android:configChanges="keyboardHidden|orientation|screenSize">
    При этом отслеживать изменение ориентации следует по событию APP_CMD_WINDOW_REDRAW_NEEDED (событиями APP_CMD_CONFIG_CHANGED и APP_CMD_CONTENT_RECT_CHANGED лучше не пользоваться, т.к. при получении этого события функции класса Window GetWidth() и GetHeight() могут возвращать старые размеры экрана).

void Application::CommandHandler ( android_app *  _pApp,
Command_t  _Command 
)
staticprivatenoexcept

Функция вызова сигнала для полученной команды.

  • Если при вызове сигнала произойдет исключение, будет вызван сигнал с идентификатором APP_CMD_ERROR, в качестве параметра которому будет передано описание произошедшего исключения).
  • При первом старте, а также повторном старте после прерывания главной кнопкой программа (в обоих случаях заново запускается функция androi_main()) получает сообщения:
    • APP_CMD_START
    • APP_CMD_RESUME
    • APP_CMD_INPUT_CHANGED
    • APP_CMD_INIT_WINDOW
    • APP_CMD_CONTENT_RECT_CHANGED
    • APP_CMD_GAINED_FOCUS
    • APP_CMD_WINDOW_REDRAW_NEEDED
  • При активации и разблокировке устройства:
    • APP_CMD_START
    • APP_CMD_RESUME
    • APP_CMD_GAINED_FOCUS
  • При отключении экрана:
    • APP_CMD_PAUSE
    • APP_CMD_SAVE_STATE
    • APP_CMD_STOP
    • APP_CMD_LOST_FOCUS
  • При нажатии главной кнопки во время работы:
    • APP_CMD_PAUSE
    • APP_CMD_LOST_FOCUS
    • APP_CMD_SAVE_STATE
    • APP_CMD_STOP - после этого возникает ошибка рисования
  • При перекрытии программы другой программой:
    • APP_CMD_PAUSE - после этого возникает ошибка рисования
    • APP_CMD_LOST_FOCUS
    • APP_CMD_TERM_WINDOW
Аргументы
[in]_pAppНе используется.
[in]_CommandКоманда, полученная приложением (см. android_native_app_glue.h, перечисление, начинающееся с APP_CMD_INPUT_CHANGED).
Необходимо сделать:
"Работу с сигналами вынести в отдельный класс, который использовать здесь" ;
template<class TWindow >
void covellite::Application::CreateMainWindow ( void  )

Функция создания главного окна программы.

  • Функция должна быть вызвана в классе главного объекта программы в функции обработки сообщения APP_CMD_INIT_WINDOW.
  • Функция создает объект главного окна программы класса, указанного в качестве параметра шаблона.
Параметры шаблона
TWindowКласс главного окна программы (должен быть наследником от класса covellite::Window).
Исключения
std::exception
  • Действие невозможно (подробнее см. описание исключения).

void Application::Run ( bool  _IsContinuousDrawing)
noexcept

Функция запуска главного цикла программы.

  • Обеспечивает извлечение и обработку получаемых программой сообщений APP_CMD_INIT_WINDOW, APP_CMD_TERM_WINDOW и т.д.).
  • Появление события приведет к вызову функции CommandHandler(), которая вызовет соответствующий сигнал.
Аргументы

false - вызов сигнала обновления окна программы только после полученного события.
