Разработка для Android на С++

Описание приводится на основе экпериментов на устройствах:

  • Android 4.0.4 (API 15, сборка для ARM (armeabi-v7a)).
  • Android 8.0.0 (API 26, сборка для ARM64 (arm64-v8a)).

Окружение

  • Android SDK.
  • Android NDK.
  • JDK.
  • Apache Ant
Заметки
  • Все внешние инструменты ставить x64 версии.
  • Пути к папкам инструментов должны состоять из символов латиницы без пробелов.
  • Настройка Android SDK производится запуском утилиты SDK Manager.exe в корневой папке SDK.

Версии используемых для экспериментов инструментов

  • Visual Sudio: 2019 (v16.4.0)
  • Android compiler: Android clang version 5.0.300080 (based on LLVM 5.0.300080)
  • Android SDK build tools: 29.0.2
  • Android SDK platform: android-29
  • Android NDK: android-ndk-r16b
  • JDK: 1.8.0_144
  • Apache Ant: 1.9.3

Однако при сборке с такими настройками при запуске на устройстве с API15 запуск программы завершится с ошибкой 'библиотека libc++_shared.so не найдена'. Чтобы обойти эту проблему, необходимо в коде java класса главной activity загрузить ее явным образом:

package com.example;
public class MainActivity extends com.alicorn.MainActivity
{
static
{
System.loadLibrary("c++_shared");
}
}
Заметки
  • Использовалась сборка для API19 (последняя, на которой доступен только 32-хбитный код), как наиболее универсальная; полученная сборка работает на устройстве с Android 4.0.4 (API15).
  • Использовался Android NDK r16b, идущий в комплекте с Visual Studio, т.к. для всех последующих полученная программа при запуске пытается загрузить dll, которой нет в системе.
  • Библиотека unwind добавлена для того, чтобы можно было использовать исключения (без нее проект соберется, но при возникновении исключения программа упадет).
  • Стандартная библиотека c++_shared используется из-за того, что в противном случае проект не собирается из-за отсутствующих реализаций функций).

Заметки
  • Использовалась сборка для API21 (первая, на которой стала доступна сборка 64-хбитного кода).
  • Проверялась сборка и запуск на Android NDK вплоть до r20b включительно, все собирается и работает без проблем.
  • Включение библиотеки unwind и явная загрузка libc++_shared.so не требуется.
  • Стандартная библиотека c++_shared используется из-за того, что в противном случае проект не собирается из-за отсутствующих реализаций функций).
