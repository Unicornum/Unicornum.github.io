Разработка для Android на С++

Исследование проводилось в расчете на использование Samsung Galaxy S Duos:

  • Android 4.0.4 (API 15).
  • Сборка для ARM (armeabi-v7a).

Окружение

  • Android SDK (устанавливается вместе с Android Studio); путь не должен содержать пробелов, установить переменную окружения ANDROID_SDK как путь к папке расположения.
  • Android NDK (путь не должен содержать пробелов, установить переменную окружения ANDROID_NDK как путь к папке расположения).
  • JDK (1.6 или 1.7); установить переменную окружения JAVA_HOME как путь к папке расположения.
Заметки
Все внешние инструменты ставить x86 версии.
Настройка Android SDK производится запуском утилиты SDK Manager.exe в корневой папке SDK (нужно будет докачать содержимое раздела Android 4.0.3 (API 15)).
При использовании Android SDK потребуется скопировать папку arm-linux-androideabi-4.9 и переименовать ее в arm-linux-androideabi-gcc4.9.
Вместо Android NDK можно использовать CrystaX (в качестве ANDROID_NDK указать путь к папке CrystaX), но в этом случае требуется добавить путь к папке libcrystax в дополнительные зависимости линковщику (см. подробности у конкретных библиотек/инструментов).

Тестировалось на:

  • Android Studio 2.1.
  • Android NDK r12.
  • JDK 1.7 r79.
  • CrystaX 10.3.1

Предопределенные макросы

  • __ANDROID__ - общий макрос признака, что используется код для Android.
  • __ANDROID_API__ - номер текущего используемого API, например, 15.

CrystaX

  • Для использования CrystaX добавить в свойствах проекта
    Компоновщик -> Общие -> Дополнительные каталоги библиотек
    $(VS_NdkRoot)/sources/crystax/libs/$(TargetArchAbi)
    (после этого программа будет компилироваться (даже Clang'ом)).
  • Также нужно, чтобы libcrysto.so попала в .apk файл (для этого нужно создать проект, в события после сбоки которому добавить копирование
    copy /Y $(VS_NdkRoot)\sources\crystax\libs\$(TargetArchAbi)\libcrystax.so
    и добавить его в Ссылки проекту *.Packaging).

Для статической сборки с crystax добавить
APP_LIBCRYSTAX := shared
в файл build-all.mk (при этом программа собирается, но не запускается...).
