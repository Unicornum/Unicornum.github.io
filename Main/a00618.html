Covellite++ v2.3

Кроссплатформенный фреймворк для разработки приложений на С++, позволяющий абстрагироваться от платформы (Windows/Android) и графического API (DirectX/OpenGL) и использующий libRocket для реализации GUI.

Возможности:

  • Компонентная система рендеринга 2D и 3D объектов, позволяющая строить сцену из набора элементарных 'кубиков', используя одни и те же данные (шейдеры, текстуры, буферы различных типов) множество раз без дублирования информации в памяти.
  • Компоненты настроек графического конвеера.
  • Ортографическая и перспективная камеры.
  • Рендеринг во внеэкранную поверхность (мультитаргет), включая буфер глкубины.
  • Чтение данных из текстуры внеэкранной поверхности.
  • Использование унифицированных HLSL/GLSL вершинных и пиксельных шейдеров.
  • Настраиваемый константный буфер для передачи данных шейдеру.
  • Поддержка PBR-текстурирования.
  • Статический/динамический вершинный буфер.
  • Статическая/динамическая трансформация (положение/ориентация/масштаб) объекта (включая возможность использовать билборда).
  • Рендеринг одиночных объектов с использованием индексного буфера.
  • Рендеринг большого количества одинаковых объектов (инстансинг; включая динамический инстанс-буфер).
  • Вызов функции обратного вызова с текущим временем работы программы.
  • Отрисовка GUI (аналог HTML/CSS) поверх кадра.
  • Сквозная система генерации и подписки на события с использованием функций обратного вызова.

Поддерживаются шейдерные реализации:

  • DirectX11/OpenGL для Windows.
  • OpenGLES3 для Android.

Документация Covellite++.

Тестовое демо:

  • Сборка для Windows x64
    • Распаковать архив, запускать файл ./Bin/x64/Example.exe
    • Вместо кнопки Back Android'a использовать Alt + Left.
  • Сборка для Android ARM/ARM64, API15+
    • Скопировать .apk файл на карту устройства, выбрать в папке, следовать указаниям установщика пакетов.

Видео

Перемещение по бесконечному, случайно генерируему ландшафту.

Рендеринг сцены, содержащей большое число точечных источников света.

Демонстрация использования PBR-материалов.

Рендеринг большого числа анимированных объектов.

Демонстрация рендеринга во внеэкранную поверхность.

2011

Unicorn 3D Engine (v2.0)

Общее описание

3D движок общего назначения.

Поддерживаемые возможности:

  • Операционная система: Windows.
  • Аппаратный API: DirectX9.
  • Создание и настройка сцены (вплоть до дефолтных настроек рендеринга) производится во внешних ресурсах (без перекомпиляции ядра).
  • Рендеринг, расчитанный на огромные открытые пространства.
  • Рендеринг во внеэкранную поверхность.
  • Несколько камер в сцене.
  • Базовые примитивы.
  • Вертексные и пиксельные шейдеры.
  • Освещение (точечное, направленное, ambient).
  • Culling.
  • Occlusion culling.
  • Скайбоксы.
  • Террайн неограниченного размера.
  • Многоступенчатое настраиваемое динамическое упрощение мешей при их удалении от камеры.
  • Скрипты (Lua), в том числе создание/уничтожение объектов в сцене.
  • Система частиц с эмуляцией физического взаимодействия.
  • Рендеринг текста в текстуру в html формате.
  • Манипуляции с объектами сцены при помощи мыши с использованием ограничивающих объемов (сфера или параллелепипед).

Скачать тестовое демо (7.55 Мб): Unicorn3DEngine

Скриншоты

2009

Единорог

Описание игры

Жанр: логическая игра на шахматной доске
API: C++, DirectX9, MFC
Программирование, дизайн и 3D модели: угу :ф)
Музыка главного меню: Евгений Иванов (aka Megaman)
Многие спасибы: раздел "Авторы" в игре

Цель игры:

При помощи охотников (которые ходят как шахматные слоны) поймать Единорога в ловушку (заблокировать его, он ходит как шахматный конь).
Более подробное описание правил игры и управления есть в самой игре в разделе "Правила".

Управление:

  • Левая кнопка мыши - выделение охотника и указание клетки, куда его нужно переместить.
  • Правая кнопка мыши - вращение/наклон доски.
  • Колесико мыши - приближение/удаление доски.

Дополнительные возможности:

  • Игру можно сохранить/загрузить в любой момент;
  • Можно отменить последний ход (вплоть до стартовой позиции или последней загрузки сохраненной игры);
  • Можно менять размер доски (от 6х6 до 12х12) и количество охотников (это влияет на сложность игры).
