Разработка для Android на С++

Исследование проводилось в расчете на использование Samsung Galaxy S Duos:

  • Android 4.0.4 (API 15).
  • Сборка для ARM (armeabi-v7a).

Окружение

  • Android SDK - устанавливается вместе с Android Studio (путь не должен содержать пробелов).
  • Android NDK (путь не должен содержать пробелов).
  • JDK.
Заметки
Все внешние инструменты ставить x86 версии.
Настройка Android SDK производится запуском утилиты SDK Manager.exe в корневой папке SDK (нужно будет докачать содержимое раздела Android 4.0.3 (API 15)).

Тестировалось на:

  • Android Studio 2.1 (Android NDK r12).
  • JDK 1.8 r101.
  • CrystaX 10.3.2 (эквивалент Android NDK r10e)

Предопределенные макросы

  • __ANDROID__ - общий макрос признака, что используется код для Android.
  • __ANDROID_API__ - номер текущего используемого API, например, 15.

CrystaX

  • Используется вместо Android NDK.
  • Работает сборка только при помощи GCC4.9.
Заметки
CrystaX требует, чтобы wchar_t была размером 4 байта.

Для статической линковки с CrystaX:

  • Добавить в свойствах проекта ("событие перед сборкой") копирование файла libcrystax.a (простое добавление пути к файлу не поможет!), например:
    copy /B /Y $(VS_NdkRoot)\sources\crystax\libs\$(TargetArchAbi)\libcrystax.a $(TargetDir)
  • В свойствах компоновщика "Дополнительные каталоги библиотек" добавить папку результата копирования ($(TargetDir)).
  • В Компоновщик -> Ввод -> Зависимости библиотек добавить crystax.

Альтернативный вариант использования CrystaX

Можно использовать CrystaX совместно с Android NDK - это позволит получить доступ к возможностям, которые отсутствуют в последнем (например, поддержку локалей):

  • Установить в настройках Visual Studio путь к Android NDK.
  • Скопировать файл libcrystax.a в Externals.
  • В свойствах компоновщика в "Дополнительные каталоги библиотек" добавить папку, в которую скопирована библиотека.
  • В Компоновщик -> Ввод -> Зависимости библиотек добавить crystax.
